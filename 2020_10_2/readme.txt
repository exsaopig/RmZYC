1. group说明：
	User-
		|-main 主函数
		|-stm32f4xx_it.c 中断处理函数
		|-其他为cube生成外设驱动，不建议修改（除串口波特率）
	UserBsp-
		|-bsp_xxx:适用于战车的底层驱动适配
	Information:
		|-xxx_info:关于xxx的数据处理
	AppCtrl-
		|-xxx_task:xxx的控制系统
		|-sys_config.h:一些系统初始化宏定义
	Debug：
	    |-用于写调试程序
	More-
		|-用于英雄等拓展功能
	
2. 外设使用说明:
    gpiof14  -LED1_Pin -绿灯
    gpioe7   -LED2_Pin -红灯  
	gpioA1   -激光
	can1     -底盘、云台、拨弹电机
	uart1    -遥控器接收机
	uart6    -TX、妙算
	uart2    -蓝牙-printf
	TIM2_CH1 -弹仓盖舵机-PA0
	TIM1_CH1 -左摩擦轮-PA8
	TIM1_CH4 -右摩擦轮-PE14
	TIM3_CH1 -蜂鸣器
	TIM3_CH2 -mpu6500温度-PB5
	spi5     -mpu6500

             2. 暂时解决了陀螺仪偏差的问题
			   （解决办法：1. 原来mpu_data.az_offset为16位，初始积分后大于65535造成初始偏差积分错误。
			              2. 实验得偏差的主要原因是q3的变化，不动时q3增量在0左右变化，
				  		     有时正的多有时负的多，总体积分出现一直递增或递减，本次直接进行一个死区限制，小于0.000005时记0.
						  3. 实验得偏差的次要原因是q3归一化时系数的变化，归一化系数在1左右变化，本次直接归1.
						  4. imu更新周期调到了1ms）
		      （安全隐患：1. 上述操作可能导致解算角度不够精确，但debug时没有察觉到。）
              （更好建议：滤波）
			3. 解决了整理版程序不能开摩擦轮的bug
            4. 进行了底盘功率限制的尝试，但没有成功。
